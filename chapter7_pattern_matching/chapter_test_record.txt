1. Regexオブジェクト生成関数
A : re.compile()

2. Regexオブジェクト生成時にraw文字列がよく使われる理由
A : バックスラッシュ症候群を防ぐ
バックスラッシュのエスケープが必要なくなるから

3. search()メソッドは何を返すか
A : Regexオブジェクトに記載した正規表現に最初に一致した文字列を含むMatchオブジェクト
Matchオブジェクト

4. Matchオブジェクトからマッチした文字列を取得する方法
A : groupメソッドを使用

5. r'(\d{3})-(\d{3}-\d{4})' について、グループに格納されている値はどれに対応するか
A : 添字の 0 が 全体、1 が 前半、2 が 後半

6. 正規表現で 「.」 と 「()」 は特殊な意味をもつ。意味のない文字として扱いたい場合、どうする？
A : バックスラッシュとセットで記述する
        → エスケープ処理

7. findall() メソッドは、文字列のリストを返す。文字列ではなく、文字列のタプルが返ってくるための条件は？
A : Regexオブジェクトを作成時、丸かっこを用いて正規表現をグループ化していること

8. 正規表現における「|」の意味は？
A : 論理和、or、または
いずれか一方とマッチすること

----9. 正規表現における「?」の意味は？
----A : 任意の一文字、
直前のパターンの0回もしくは1回のマッチ ---つまり---> 非貪欲モードでマッチ（複数パターンのマッチ方法がある場合、末尾につけておくと一番少ないマッチの仕方で出力される）

10. 「+」と「*」の違い
A : 直前に記載の正規表現を繰り返す回数が１回以上か０回以上か

11. 「{3}」、「{3,5}」の違い
A : パターンの繰り返し回数が３回、繰り返し回数が３から５回のうち任意の回数

12. 「\d」「\w」「\s」
A : 半角数字、半角英数（大小どちらも可）、空欄

13. 「\D」「\W」「\S」
A : 「12.」で記載の内容の否定形

14. 大文字と小文字を区別しないようにする方法
A : re.IGNORECASE をRegexオブジェクト生成時に記載

------15. 「.」は何を表すか？また、re.DOTALL の際、「.」は何を表すか？
------A : 任意の１文字を示すが、re.DOTALLがあると、ただのドットとして扱う
改行文字以外の任意の一文字、re.DOTALLがあると改行文字にも対応する

---------16. 「.*」と「.*?」の違いは？
-------A : 任意の0文字以上の文字列、
貪欲マッチ、被貪欲マッチ

17. 全ての数字と小文字にマッチする文字集合の記載方法は？
A : [0-9a-z]

18. 「 num_regex = re.compile(r'\d+') 」 に対し、num_regex.sub('X', '12 drummers, 11 pipers, five rings, 3 hens') は何を返すか？
A : 'X drummers, X pipers, five rings, X hens'

------19. re.VERBOSE を渡すと何ができるか
------A : 正規表現を複数行に渡って記述でき、コメントを加えることもできる
空白が許される、途中でコメントをかける

20. 「三桁ごとにカンマがつく数字」にマッチする正規表現を書く
re.compile(r'''
  \d{1-3}               # 大きい桁部分の数字
  (,\d{3})*             # カンマ区切りとより小さい桁部分
''', re.VERBOSE)

21. Nakamoto姓の方のフルネームを出力する正規表現
re.compile(r'''
  ([A-Z]{1}[a-z]+)      # 名前          -> {1}いらない 
  \s                    # 間を開ける
  (Nakamoto)            # 苗字
''', re.VERBOSE)

22. 最初がAlice Bob Carol、二番目の単語がeats pets throws、三番目の数がapples cats baseballsでピリオド終了する文の正規表現
re.compile(r'''
  (Alice|Bob|Carol)\s
  (eats|pets|throws)\s
  (apples|cats|baseballs)\.
''', re.VERBOSE)                                -> re.IGNORECASE 忘れ
